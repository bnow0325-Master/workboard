# BNOW Workboard 개발 지침

## 제품 목적

Workboard는 BNOW 내부 프로젝트와 담당자별 업무, 댓글, 확인·승인요청을 관리하는 사내 운영도구임.

## 작업 원칙

1. 작업 전에 원격 `main`의 최신 변경사항을 확인함.
2. `agent/<작업명>` 기능 브랜치에서 변경하고 PR로 병합함.
3. 현재 단일 `index.html` 구조에서는 변경 범위를 작게 유지함.
4. 새 기능을 추가할 때 전역변수와 인라인 이벤트 핸들러를 더 늘리지 않도록 함.
5. 실제 직원 이메일, 이름, 업무내용, 고객·농가정보를 새 테스트 데이터로 커밋하지 않음.
6. Supabase `service_role` 키, DB 비밀번호, Webhook URL·토큰을 브라우저 코드에 넣지 않음.
7. 관리자 권한을 클라이언트의 이메일 비교만으로 판단하지 않음.
8. Supabase 테이블 변경 시 Row Level Security 정책을 함께 검토함.
9. 사용자 입력은 HTML 출력 전에 이스케이프하고, 외부 URL은 허용 목록으로 제한함.
10. 기능 변경 후 로그인, 프로젝트 CRUD, 댓글, 승인요청, 모바일 화면을 확인함.

## 개선 우선순위

1. 저장소 비공개 전환
2. 관리자 권한 서버 검증
3. Webhook·알림 전송 서버 이전
4. HTML·CSS·JavaScript 파일 분리
5. 자동 테스트 추가

## 완료 기준

- GitHub Actions 정적 검증 통과
- 브라우저 콘솔 오류 없음
- 관리자·일반 사용자 권한 차이 확인
- 모바일 레이아웃 확인
- 개인정보·비밀정보 노출 여부 확인
